# Progressive Disclosure ガイドライン

人間やAIが一度に処理できる情報量には限界があります。また、LLMのコンテキストウィンドウの最適化のため、「漸進的開示 (Progressive Disclosure)」の原則を適用します。

## 1. 原則

1. **メインプロンプトの圧縮**: `SKILL.md` の行数は **理想で300行以内、超えても500行以内** に収めること。
2. **参照パターンの活用**: 詳細な手順、長大なスキーマ定義、環境設定の手引きなどは `references/*.md` や `scripts/*.py` などの独立したファイルに切り出す。
3. **明示的なポインター**: `SKILL.md` から切り出したファイルへは、Markdownのリンク、または指示文として「詳細な制約については `references/xyz.md` を参照せよ」と明記する。

## 2. 分割の閾値とアプローチ

- 1つのセクションブロック（例: `## 構成タスク`）が100行を超える場合、独立した Markdown に切り出す。
- `SKILL.md` での定義は、全体の **処理の流れ（オーケストレーション）** に特化させる。
- サブプロンプトや、プロンプトに組み込むと長過ぎる固定データ（テンプレート等）は、必ず `assets/` または `templates/` フォルダから読み込ませるか、Python等のスクリプト側で処理させる。

## 3. 実装例

**SKILL.md (悪い例 - すべて詰め込んでいる)**
```markdown
# My Skill
...
## データスキーマ
（500行のJSONスキーマ定義がドーンと記載されている）
...
## 手順
1. スキーマに従って...
```

**SKILL.md (良い例 - Progressive Disclosure)**
```markdown
# My Skill
...
## データスキーマと手順
1. `references/schema_def.md` のデータ構造を読み込み、理解してください。
2. その後、同ドキュメントに記載された手順1〜5を実行してください。
```
