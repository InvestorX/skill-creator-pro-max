# 美学制約 (Aesthetics Constraints)

`frontend_ui` を実装する際は、以下のルールを厳格に適用し、プレミアムで洗練されたユーザー体験を提供すること。

## 1. CSS Framework
- **Tailwind CSS を標準フレームワークとして使用する**（CDN経由での導入を推奨）。
- インラインのスタイル指定は避け、Tailwindのユーティリティクラスを活用すること。

## 2. Typography
- Arial等の一般的なシステムフォントの使用は避ける。
- **Google Fonts** からモダンで特徴的なフォントをインポートして使用する（推奨: `Inter`, `Space Grotesk`, `Outfit`, `Sora`）。
- タイトル、本文、アクセントで明確なフォントウェイトのコントラスト（例: 300, 500, 800等）を付ける。

## 3. Color & Theme
汎用的な色パレットを避け、以下のいずれかのテーマを採用すること：
1. **Solarpunk**: 自然と技術の調和（グリーン系＋ウォームライト）
2. **Neo-brutalism**: 高コントラスト、太いボーダー、原色使い
3. **Glassmorphism**: ぼかしバックグラウンド（backdrop-blur）、半透明のカード、鮮やかな背景グラデーション
4. **Cyberpunk（攻殻機動隊風）**: 黒/濃紺背景に、サイバーパンク風のネオングリーン/シアン、グリッドパターン

## 4. Effects
- 状態変化（ホバー、フォーカス）時には、必ず **スムースなトランジション効果**（`transition-all duration-300` 等）を含める。
- **グリッチエフェクト**: 「Cyberpunk」テーマを採用した場合、タイトルや一部のアクセントテキストに対してCSSによるグリッチエフェクト（テキストのランダムシフト、RGB分離、スキャンライン）を自動適用すること。

## 5. 感情報酬型タスク過多ユーザー対応ガイドライン（ペルソナ対応）
タスク過多で疲弊したユーザーの認知負荷を下げるため、以下のデザイン原則を適用：
- **余白ファースト**: 情報密度を意図的に下げ、Padding/Marginを広めに取る（`p-8`, `gap-6` など）。
- **番号選択・ワンクリック**: フリーテキスト入力を極力減らし、選択式のUIを基本とする。
- **柔らかなトーン**: システムメッセージは命令形を避け、丸みのあるボタンUI（`rounded-full`, `rounded-2xl`）を採用する。
- **マイクロ承認**: アクション成功時に、緑色のチェックマークアニメーションなど「小さな達成感」を即座にフィードバックする。
- **段階的開示**: 一度に全ての設定を見せず、アコーディオン (`<details>`) を使って情報を隠す。
