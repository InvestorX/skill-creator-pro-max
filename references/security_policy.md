# セキュリティポリシー (Security Policy)

生成するスキルにおいて、以下のセキュリティ基準を遵守する。

## 1. クレデンシャルの管理
- APIキー、トークン、パスワード、接続文字列などのハードコードは **一切禁止**。
- 全ての機密情報は、システム環境変数または `.env` ファイルから読み込む設計とする。
- スキルパッケージ生成時には、ユーザーへの案内用として必ず `.env.example` ファイル（ダミー値入り）を同梱する。

## 2. 破壊的操作の保護
- ファイルの削除、APIを通じたリソースの変更、デプロイ操作などを行うスクリプトは、デフォルトで **`--dry-run` モード** （安全に結果のみを出力するモード）で動作するよう構成する。
- 本番実行の際は、必ずユーザーによる明示的なフラグ指定（例: `--force`, `--execute`）がない限り、実行をブロックする制御を組み込む。

## 3. 入力値の検証
- ユーザー入力値やAPIからの外部入力値を用いてコマンド実行（`subprocess`, `os.system` 等）を行う場合、シェルインジェクションを防ぐため、サニタイズを行うこと。
- SQLを実行する場合（`backend_api`）は、必ずプレースホルダー（プリペアドステートメント）を使用する。

## 4. WebアクセスとCORS
- Chrome拡張機能などで外部APIへアクセスする場合は、必要最小限の `host_permissions` を `manifest.json` に設定する。
- `frontend_ui` での通信において、ユーザーのセッションやトークンをLocalStorage等へ安易に保存しないアーキテクチャを推奨する。
